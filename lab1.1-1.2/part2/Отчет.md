#### **Файл: `app/blog/[slug]/page.tsx` (Динамическая страница статьи)**

```tsx
import { notFound } from 'next/navigation'
import Link from 'next/link'
import { blogPosts } from '../data'

// Функция для генерации статических путей (SSG)
export async function generateStaticParams() {
  return blogPosts.map((post) => ({
    slug: post.slug,
  }))
}

export default function BlogPostPage({
  params,
}: {
  params: { slug: string }
}) {
  const post = blogPosts.find(p => p.slug === params.slug)
  
  if (!post) {
    notFound()
  }

  return (
    <article className="max-w-3xl mx-auto">
      <header className="mb-8">
        <h1 className="text-3xl font-bold mb-2">{post.title}</h1>
        <div className="flex justify-between text-gray-600">
          <span>{post.date}</span>
          <span>Автор: {post.author}</span>
        </div>
      </header>
      
      <div className="prose max-w-none">
        <p className="text-lg mb-4">{post.excerpt}</p>
        <p className="text-gray-700">{post.content}</p>
      </div>
      
      <div className="mt-8 pt-4 border-t">
        <Link 
          href="/blog"
          className="text-blue-600 hover:text-blue-800"
        >
          ← Вернуться к списку статей
        </Link>
      </div>
    </article>
  )
}
```

## **Ответы на вопросы**

### **Часть 1: React-приложение**

**1. Объясните принцип работы хука `useState`**

`useState` — это хук React, который позволяет добавлять состояние к функциональным компонентам. Он возвращает массив из двух элементов: текущее значение состояния и функцию для его обновления. При вызове функции обновления React перерисовывает компонент с новым значением состояния.

**2. Почему в React важно использовать неизменяемое состояние?**

Неизменяемое (immutable) состояние важно по следующим причинам:
- React использует поверхностное сравнение объектов для определения необходимости перерисовки
- Мутация состояния напрямую может привести к непредсказуемому поведению
- Неизменяемость упрощает отладку и отслеживание изменений
- Позволяет использовать функции оптимизации like `React.memo`

**3. Какой метод массива вы использовали для удаления задачи и почему?**

Для удаления задачи использован метод `filter`. Он создаёт новый массив, включая только элементы, удовлетворяющие условию. Это обеспечивает:
- Неизменяемость исходного массива
- Простоту и читаемость кода
- Соответствие принципам React

**4. В чём преимущества TypeScript при разработке React-приложений?**

Преимущества TypeScript:
- Статическая типизация обнаруживает ошибки на этапе разработки
- Автодополнение и навигация по коду в IDE
- Самодокументируемость кода через типы
- Упрощает рефакторинг
- Лучшая поддержка в больших командах

### **Часть 2: Next.js**

**1. Что такое SSG (Static Site Generation) и как он реализован в вашем проекте?**

SSG (Static Site Generation) — это метод генерации статических HTML-страниц на этапе сборки проекта. В моём проекте SSG реализован через функцию `generateStaticParams`, которая определяет все возможные пути для динамических страниц блога:

```typescript
export async function generateStaticParams() {
  return blogPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

**2. Как работает файловая маршрутизация в Next.js?**

В Next.js используется файловая маршрутизация на основе структуры папок:
- `app/page.tsx` → `/`
- `app/about/page.tsx` → `/about`
- `app/blog/page.tsx` → `/blog`
- `app/blog/[slug]/page.tsx` → `/blog/<любой-slug>`

**3. Какие преимущества даёт использование `generateStaticParams`?**

`generateStaticParams`:
- Позволяет предварительно сгенерировать все статические страницы
- Улучшает производительность за счёт статического рендеринга
- Обеспечивает лучшее SEO
- Сокращает время загрузки страниц

**4. В чём разница между `npm run dev` и `npm run build`?**

- `npm run dev` — запуск сервера разработки с горячей перезагрузкой (HMR)
- `npm run build` — создание оптимизированной production-сборки с минификацией и оптимизацией
