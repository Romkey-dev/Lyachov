## **3. Исходный код**

### **Файл: `src/App.tsx`**

```tsx
import React, { useState } from 'react';

// Тип для задачи
interface Task {
  id: number;
  text: string;
  completed: boolean;
}

function App() {
  // Состояние для списка задач
  const [tasks, setTasks] = useState<Task[]>([
    { id: 1, text: 'Изучить React', completed: true },
    { id: 2, text: 'Написать To-Do приложение', completed: false }
  ]);

  // Состояние для новой задачи
  const [newTask, setNewTask] = useState('');

  // Функция добавления задачи
  const addTask = () => {
    if (newTask.trim() === '') return;
    
    const task: Task = {
      id: Date.now(),
      text: newTask,
      completed: false
    };
    
    setTasks([...tasks, task]);
    setNewTask('');
  };

  // Функция удаления задачи (Реализовано самостоятельно)
  // Использует метод filter для создания нового массива без удаляемой задачи
  const removeTask = (id: number) => {
    setTasks(tasks.filter(task => task.id !== id));
  };

  // Функция переключения статуса задачи (Реализовано самостоятельно)
  // Использует метод map для обновления массива задач
  const toggleTask = (id: number) => {
    setTasks(tasks.map(task => 
      task.id === id ? { ...task, completed: !task.completed } : task
    ));
  };

  // Подсчёт выполненных задач
  const completedCount = tasks.filter(task => task.completed).length;

  return (
    <div className="min-h-screen bg-gray-100 p-8">
      <div className="max-w-2xl mx-auto bg-white rounded-lg shadow-lg p-6">
        <h1 className="text-3xl font-bold text-center mb-6 text-gray-800">
          📝 Список задач
        </h1>
        
        {/* Форма добавления задачи */}
        <div className="flex gap-2 mb-6">
          <input
            type="text"
            value={newTask}
            onChange={(e) => setNewTask(e.target.value)}
            onKeyDown={(e) => e.key === 'Enter' && addTask()}
            placeholder="Введите новую задачу..."
            className="flex-grow px-4 py-2 border border-gray-300 rounded-lg focus:outline-none focus:ring-2 focus:ring-blue-500"
          />
          <button
            onClick={addTask}
            className="px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition"
          >
            Добавить
          </button>
        </div>

        {/* Список задач */}
        <div className="space-y-3">
          {tasks.map(task => (
            <div 
              key={task.id} 
              className="flex items-center justify-between p-3 bg-gray-50 rounded-lg border"
            >
              <div className="flex items-center gap-3">
                <input
                  type="checkbox"
                  checked={task.completed}
                  onChange={() => toggleTask(task.id)}
                  className="h-5 w-5 text-blue-600"
                />
                <span className={`${task.completed ? 'line-through text-gray-500' : 'text-gray-800'}`}>
                  {task.text}
                </span>
              </div>
              
              {/* Кнопка удаления с иконкой */}
              <button
                onClick={() => removeTask(task.id)}
                className="px-3 py-1 text-red-500 hover:text-red-700 hover:bg-red-50 rounded transition"
              >
                ✕ Удалить
              </button>
            </div>
          ))}
        </div>

        {/* Статистика */}
        <div className="mt-6 pt-4 border-t">
          <p className="text-gray-600">
            Всего задач: {tasks.length}
          </p>
          <p className="text-gray-600">
            Выполнено: {completedCount} из {tasks.length}
          </p>
        </div>

        {/* Сообщение при пустом списке */}
        {tasks.length === 0 && (
          <div className="text-center py-8 text-gray-500">
            <p>Список задач пуст</p>
            <p className="text-sm">Добавьте первую задачу!</p>
          </div>
        )}
      </div>
    </div>
  );
}

export default App;
```

## **4. Реализованные функции**

### **A. Удаление задач (removeTask)**

```typescript
const removeTask = (id: number) => {
  setTasks(tasks.filter(task => task.id !== id));
};
```

**Пояснение:** Используется метод `filter` для создания нового массива, который не содержит задачу с указанным ID. Это соответствует принципу неизменяемого состояния (immutability) в React.

---

### **B. Переключение статуса задачи (toggleTask)**

```typescript
const toggleTask = (id: number) => {
  setTasks(tasks.map(task => 
    task.id === id ? { ...task, completed: !task.completed } : task
  ));
};
```

**Пояснение:** Используется метод `map` для создания нового массива с обновлённым статусом задачи. Для найденной задачи статус `completed` инвертируется.

---

**Пояснение:** Добавлена кнопка удаления с иконкой ✕ и текстом "Удалить". Стилизована для лучшего UX: красный цвет, hover-эффекты.

## **6. Ответы на вопросы**

### **1. Объясните принцип работы хука `useState`**

`useState` — это хук React, который позволяет добавлять состояние к функциональным компонентам. Он возвращает массив из двух элементов:
- Текущее значение состояния
- Функцию для его обновления

При вызове функции обновления React перерисовывает компонент с новым значением состояния. Инициализация происходит один раз при первом рендере.

---

### **2. Почему в React важно использовать неизменяемое состояние?**

Неизменяемое (immutable) состояние важно по следующим причинам:

1. **React использует поверхностное сравнение** объектов для определения необходимости перерисовки компонента.

2. **Мутация состояния напрямую** может привести к непредсказуемому поведению, так как React не обнаружит изменений.

3. **Неизменяемость упрощает отладку** и отслеживание изменений в DevTools.

4. **Позволяет использовать функции оптимизации** like `React.memo`, `useMemo`, `useCallback`.

5. **Поддержка отката изменений** (time-travel debugging) в Redux DevTools.

---

### **3. Какой метод массива вы использовали для удаления задачи и почему?**

Для удаления задачи использован метод `filter`:

```typescript
const removeTask = (id: number) => {
  setTasks(tasks.filter(task => task.id !== id));
};
```

**Почему filter:**
- Создаёт новый массив, не мутируя исходный
- Простой и читаемый код
- Соответствует принципу неизменяемости в React
- O(n) сложность, что приемлемо для небольших списков

Альтернативы: `splice` (мутирует массив), `reduce` (сложнее).
