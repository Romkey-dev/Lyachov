## 3. Реализация

### 3.1 Модели данных (models.py)

Для валидации данных использовались Pydantic модели:

```python
from pydantic import BaseModel, Field
from typing import Optional, List
from enum import Enum
from datetime import date

# Enum для жанров книг
class Genre(str, Enum):
    FICTION = "fiction"
    NON_FICTION = "non_fiction"
    SCIENCE = "science"
    FANTASY = "fantasy"
    MYSTERY = "mystery"
    BIOGRAPHY = "biography"

# Модель для создания книги
class BookCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=200)
    author: str = Field(..., min_length=1, max_length=100)
    genre: Genre = Field(...)
    publication_year: int = Field(..., ge=1000, le=2026)
    pages: int = Field(..., gt=0)
    isbn: str = Field(..., pattern=r'^\d{13}$')
```

### 3.2 Основное приложение (main.py)


### 3.3 Роутеры (routers.py)

Реализованы следующие эндпоинты:

| Метод | URL | Описание |

## 4. Тестирование API

### 4.1 Создание книги (POST /books)

**Запрос:**
```bash
curl -X POST "http://localhost:8000/api/v1/books" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Преступление и наказание",
    "author": "Фёдор Достоевский",
    "genre": "fiction",
    "publication_year": 1866,
    "pages": 671,
    "isbn": "9781234567898"
  }'
```

**Ответ:**
```json
{
  "id": 1,
  "title": "Преступление и наказание",
  "author": "Фёдор Достоевский",
  "genre": "fiction",
  "publication_year": 1866,
  "pages": 671,
  "isbn": "9781234567898",
  "available": true
}
```

### 4.2 Заимствование книги (POST /books/1/borrow)

**Запрос:**
```bash
curl -X POST "http://localhost:8000/api/v1/books/1/borrow" \
  -H "Content-Type: application/json" \
  -d '{"borrower_name": "Иван Иванов", "return_days": 14}'
```

**Ответ:**
```json
{
  "id": 1,
  "title": "Преступление и наказание",
  "author": "Фёдор Достоевский",
  "genre": "fiction",
  "publication_year": 1866,
  "pages": 671,
  "isbn": "9781234567898",
  "available": false,
  "borrowed_by": "Иван Иванов",
  "borrowed_date": "2026-04-29",
  "return_date": "2026-05-13"

```

## 5. Ответы на контрольные вопросы

### 5.1 В чем преимущества использования Pydantic моделей для валидации?

**Ответ:** Pydantic модели обеспечивают автоматическую валидацию данных на этапе их получения. Основные преимущества:
- Автоматическая проверка типов данных
- Встроенные ограничения (min_length, max_length, pattern, ge, le и др.)
- Автогенерация JSON Schema для документации
- Понятные сообщения об ошибках валидации
- Возможность использовать Python типы для аннотации

### 5.2 Как работает автоматическая документация в FastAPI?

**Ответ:** FastAPI автоматически генерирует документацию на основе:
- Pydantic моделей (описание данных, валидация)
- аннотаций типов в функциях
- docstring-ов в функциях
- аннотаций параметров (Query, Body, Path и др.)

Документация доступна по адресам:
- `/docs` — Swagger UI
- `/redoc` — ReDoc

### 5.3 Почему важно проверять уникальность ISBN?

**Ответ:** ISBN — это уникальный идентификатор книги. Проверка уникальности необходима because:
- Каждая книга в библиотеке должна иметь уникальный идентификатор
- Предотвращает дублирование записей
- Обеспечивает корректность поиска и обновления книг

### 5.4 Какие статус-коды HTTP вы использовали и почему?

| Код | Использование | Причина |
|-----|---------------|---------|
| 200 | GET (успешное получение) | Стандартный код успеха |
| 201 | POST (создание) | Ресурс успешно создан |
| 204 | DELETE (удаление) | Успешное удаление без возврата контента |
| 400 | Ошибка валидации/бизнес-логики | Неверные данные от клиента |
| 404 | Ресурс не найден | Книга с указанным ID не существует |
