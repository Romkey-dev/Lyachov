## 📋 Описание проекта

Полнофункциональный REST API для управления задачами, разработанный на **Express.js** с использованием **Node.js** и файловой системы для хранения данных. Проект демонстрирует лучшие практики разработки бэкенда, включая валидацию, обработку ошибок, фильтрацию и пагинацию.

## 💻 API endpoints (8 основных)

| Метод | Endpoint | Описание |
|-------|----------|---------|
| **POST** | `/api/tasks` | Создать задачу |
| **GET** | `/api/tasks` | Получить все (с фильтрацией) |
| **GET** | `/api/tasks/:id` | Получить одну задачу |
| **PUT** | `/api/tasks/:id` | Обновить задачу |
| **PATCH** | `/api/tasks/:id/complete` | Отметить выполненной |
| **DELETE** | `/api/tasks/:id` | Удалить задачу |
| **GET** | `/api/tasks/stats/summary` | Получить статистику |
| **GET** | `/api/tasks/search/text?q=...` | Поиск задач |

## 🔍 Примеры использования

### Создать задачу
```bash
curl -X POST "http://localhost:3000/api/tasks" \
  -H "Content-Type: application/json" \
  -d '{"title": "Купить продукты", "priority": 2}'
```

### Получить все задачи (отсортированные)
```bash
curl "http://localhost:3000/api/tasks?sortBy=-priority&limit=5"
```

### Поиск задач
```bash
curl "http://localhost:3000/api/tasks/search/text?q=продукты"
```

### Статистика
```bash
curl "http://localhost:3000/api/tasks/stats/summary"
```

## 🐛 Частые вопросы

### Q: Как запустить?
A: Выполните `npm install` затем `npm run dev`

### Q: На каком порту запускается?
A: На порту 3000 (или установите PORT в .env)

### Q: Как добавить новый endpoint?
A: Отредактируйте `src/routes/tasks.js` и добавьте `router.get(...)`

### Q: Где хранятся данные?
A: В файле `tasks.json` (создается автоматически)

### Q: Как изменить валидацию?
A: Отредактируйте `src/middleware/validation.js`

### Q: Как добавить новую категорию?
A: Добавьте в `validation.js` в поле category

### Q: Почему задача не создается?
A: Проверьте валидацию - минимум 3 символа в title

### Q: Как тестировать API?
A: Используйте curl команды из `ПРИМЕРЫ_ТЕСТИРОВАНИЯ.md`
